The Bad Side of Node

Node as a web Server

Story Time

  • It's a B2B project; our client has a few websites available in dozens of language
  • Client has multiple sites in multiple languages with same brand, different layouts
  • Client's main sites were already using DXA
  • Client has content that is displayed in screens in on-site facilities; not strictly "web content"
  • The content stored in the CMS will have variables. Those variables will get resolved in an application which manages the on-site displays

An Architecture that we tried

  • We would use DXA TBBs in Tridion
  • We would build a Rest API in Node.js that provided content
  • A third party would be responsible for managing the screen displays
  • But we had to build a site preview that showed what content would look like in those displays

Motivations

This wasn't browser content, client didn't need a web app

The client needed an API for another application

The client suggested node.js because it would appeal to their skillsets

Building a Rest API with node

This is kind of node's bread and butter here. Node loves it some Rest APIs

We created a facade for the OData microservices. Instead of OData requests, we made much more routed API requests. All using GET.
